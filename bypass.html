Overview

In many school or corporate environments, network access is filtered at the DNS level. This guide explains how to change your device's DNS settings to use our public Pi-hole server, which can help bypass certain content blocks and enhance privacy by filtering trackers.

1. Our Public Pi-hole DNS

We host our own public Pi-hole DNS resolver pre-configured with our curated blocklist. The only information you need is our server's IP address:

Our Pi-hole instance uses the blocklist maintained at our GitHub repository.

2. Applying the DNS on Your Device

Select your operating system below for detailed instructions. We'll try to detect your OS and open the correct section for you automatically.

ChromeOS

Method 1: Standard Settings (For Personal/Unmanaged Chromebooks)

  1. Click on the time in the bottom-right corner to open the Quick Settings panel, then click the Settings gear ‚öôÔ∏è.
  2. In the Settings menu, go to the Network section and click on Wi-Fi.
  3. Click the > arrow next to the name of the Wi-Fi network you are currently connected to.
  4. Scroll down and expand the Network section.
  5. Under "Name servers," select Custom name servers.
  6. In the first "Name server" field, enter our DNS IP address: 34.48.64.47. You can leave the other fields blank.

Method 2: chrome://network (For Managed/Locked-Down Chromebooks)

If the standard settings are blocked, use this method:

  1. Navigate to chrome://network in your Chrome browser.
  2. Click on Network Select at the top, then click Render Network Select.
  3. If you haven't already, connect to the network you would like to apply the bypass to by clicking on it.
  4. Click on your current WiFi network to open the dialog. If it isn't visible, click the empty area under Render Network Select.
  5. Scroll down to the Name Servers section.
  6. Change the setting from "Automatic name servers" to Custom name servers.
  7. Paste our Pi-hole DNS IP address, 34.48.64.47, into the first "Name server" box.

Note: On some devices, the DNS field might revert to `0.0.0.0` while typing. To work around this, copy the IP address first, then quickly paste it and close the dialog.

Windows

Instructions for Windows 10 & 11

  1. Press the Windows Key, type "Control Panel", and press Enter.
  2. Go to Network and Internet > Network and Sharing Center.
  3. On the left, click Change adapter settings.
  4. Right-click on your active network connection (usually "Wi-Fi" or "Ethernet") and select Properties.
  5. Select Internet Protocol Version 4 (TCP/IPv4) from the list and click the Properties button.
  6. Select the radio button for "Use the following DNS server addresses".
  7. In the "Preferred DNS server" box, enter: 34.48.64.47
  8. Leave the "Alternate DNS server" box empty. Click OK, then Close.
macOS

Instructions for macOS

  1. Open System Settings (or System Preferences on older versions).
  2. Click on Network.
  3. Select your active connection (e.g., Wi-Fi) from the list on the left, then click the Details... (or Advanced...) button.
  4. Go to the DNS tab.
  5. Under "DNS Servers," click the + button to add a new server.
  6. Enter our DNS IP address: 34.48.64.47
  7. If there are other DNS servers listed, you can select them and click the - button to remove them for best results.
  8. Click OK, then click Apply.

Linux

Instructions for Linux

Changing DNS settings on Linux varies widely depending on your distribution and desktop environment. Most modern desktop environments use NetworkManager.

For GNOME (Ubuntu, Fedora, etc.) or KDE Plasma:
  1. Open Settings and navigate to the Network or Connections section.
  2. Find your active Wi-Fi or Ethernet connection and click the gear icon ‚öôÔ∏è to edit its settings.
  3. Go to the IPv4 tab.
  4. Disable the "Automatic" toggle next to DNS.
  5. In the DNS entry field that appears, enter our server IP: 34.48.64.47
  6. Click Apply to save the changes. You may need to disconnect and reconnect to the network for the new settings to take effect.

3. MAC Address Randomization

To enhance privacy, especially on public or school networks, consider enabling MAC address randomization if your device supports it. This makes it harder to track your specific device across different Wi-Fi networks.

On most modern devices (Windows, macOS, iOS, Android), this is enabled by default or can be found in the specific Wi-Fi network's settings. For ChromeOS, you can enable it via chrome://flags by searching for "MAC address randomization" and setting it to "Enabled".
